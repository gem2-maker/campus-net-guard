# Campus Net Guard

Campus Net Guard 是一个用于**深圳技术大学北区校园网**的自动认证脚本。
当网络中断、认证会话过期、或者电脑从睡眠恢复后，它会自动检测并重新登录，避免手动反复点网页。

> ✅ **重点：可以开着梯子使用。** 脚本会先识别“是否命中校园网认证场景”，命中才触发登录；正常联网（含开着梯子时）不会无脑反复登录。

## 这个脚本解决什么问题

在校园网环境里，常见情况是：

- 网络看起来连着，但实际外网访问失败
- 认证会话到期后需要重新登录
- 电脑重启/唤醒后需要再次认证

这个脚本会周期性检查网络状态，发现“需要认证”就自动执行登录流程。

## 工作原理（通俗版）

可以把它理解成一个“值班员”：

1. **先检查网络是否正常可用**
2. 如果不可用，判断是否是“需要校园网认证”
3. 命中后，按配置好的账号参数向认证接口发起登录请求
4. 登录成功后继续待命，失败则按策略重试

核心点：不是盲目反复登录，而是“先检测，后动作”。

## 项目结构

- `campus_net_guard.py`：主程序（检测 + 自动登录）
- `campus-net-guard.json`：配置文件模板
- `run_campus_net_guard.ps1`：后台启动脚本
- `install_campus_net_routes.ps1`：校园网直连路由安装脚本（Windows）
- `install_campus_net_task.ps1`：开机自启任务安装脚本（Windows）

## 环境要求

- Windows + PowerShell
- Python 3.x（建议 3.9+）

## 快速开始（详细版）

### 1) 准备配置文件

编辑 `campus-net-guard.json`：

```json
{
  "account": "你的学号或校园网账号",
  "password": "你的校园网密码",
  "domain": "@cucc",
  "ac_id": "17"
}
```

字段说明：

- `account`：登录账号
- `password`：登录密码
- `domain`：账号后缀（按学校网络实际要求填写）
- `ac_id`：认证区域/接入点编号（北区按当前配置使用 `17`）

> 如果学校网络参数后续调整，按最新认证页面参数更新即可。

### 2) 先跑一次单次模式（建议）

```powershell
python campus_net_guard.py --once
```

用途：

- 验证配置是否正确
- 看认证流程是否走通
- 出错时便于定位问题

### 3) 再切常驻模式

```powershell
python campus_net_guard.py
```

常驻模式会持续监控网络并在需要时自动认证。

## 常见问题

### Q1：脚本在运行，但还是上不了网？

先做这几个检查：

1. 确认账号密码是否可在网页登录成功
2. 确认 `domain` 和 `ac_id` 是否与当前校区一致
3. 看是否被本机代理/VPN/防火墙拦截了认证请求
4. 用 `--once` 模式看日志里的具体报错

### Q2：为什么不用“每分钟强制登录一次”？

因为那样粗暴且容易触发风控。这个脚本是“检测到需要认证才登录”，更稳。

### Q3：会不会影响正常网络？

正常情况下不会。它只在检测到需要认证时才发登录请求。

### Q4：开着梯子（代理/VPN）还能自动校园网登录吗？

可以。脚本是“先检测再登录”，不是定时强登：

- 命中校园网认证页/重定向时，自动登录
- 非认证场景（包括你已经正常联网、或在代理/VPN链路下）不乱触发

如果你本机代理规则把校园网认证域名也代理走了，可能会影响识别；这种情况把认证相关域名设为直连即可。
